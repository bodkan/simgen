Base R plots use a single function and specify a (potentially large) number of parameters which change their ‚Äúaesthetic‚Äù properties and visual elements.

. . .


Creating a new figure (even from the same data) usually requires a completely different command.

tidyverse provides ‚Äúgrammar for data manipulation‚Äù‚Ä¶

A ‚Äúmini-language‚Äù with verbs like select(), filter(), etc.

(Helps us avoid ‚ÄúR bootcamp‚Äù-level nightmares.)

ggplot2 provides ‚Äúgrammar for visualizations‚Äù‚Ä¶

(To make visualizations easier in a similar sense.)

Layers in ‚ÄúGrammar of Graphics‚Äù

Illustration of Layers in Grammar of Graphics from Quebec Centre for Biodiversity Science

Layers in ‚ÄúGrammar of Graphics‚Äù

  • data ‚Äî our data frame
  • aesthetics ‚Äî ‚Äúmapping‚Äù of columns to visual properties of a figure (x, or y axes, color, shape, etc.)
  • geoms ‚Äî graphical elements to be plotted (histograms, points, lines, etc.).
  • many other customizations


A tiny example

A quick advertisement for the ‚Äúlayering‚Äù concept of ggplot2

What data to visualize?

ggplot(penguins)

How to visualize it? (x ‚Äúaesthetic‚Äù)

ggplot(penguins, aes(x = body_mass_g))

How to visualize it? (x ‚Äúaesthetic‚Äù)

ggplot(penguins, aes(x = body_mass_g)) +

On what dimensions?

ggplot(penguins, aes(x = body_mass_g)) + # the + is a bit like %>%
  xlim(2000, 8000)

What to visualize? (‚Äúgeom‚Äù)

ggplot(penguins, aes(x = body_mass_g)) +
  xlim(2000, 8000) +
  geom_histogram()
